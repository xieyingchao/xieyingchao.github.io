swagger学习笔记

发表于 2020-12-03 | 分类于 学习笔记

¶什么是Swagger？

​ Simplify API development for users, teams, and enterprises with the Swagger open source and professional toolset.

​ 一个简化接口文档开发的工具。通过在代码里添加Swagger注解，即可自动生成对应的API文档，不用再手动编写文档。后端开发者通过Swagger自动生成API文档供前端和其他人使用，即使后期修改了接口，只需修改对应的注解，即可修改对应的文档，避免了文档和接口不一致的情况。

¶Springfox Swagger

​ Spring官方基于Swagger规范，将Swagger整合成Springfox-Swagger项目，方便整合进Spring项目。

​ Springfox Swagger目前已经更新到3.0.0。

    配置类主要用来配置Api的一些信息，定义需要扫描的类或者方法，格式一般比较固定。

    需要创建配置类，并添加EnableSwagger2注解，开启Swagger注解模式。

    ¶apiInfo函数：

    apiInfo函数通过调用ApiInfoBuilder来定义api的一些信息，主要包括：

    • title：api文档标题
    • description：文档描述
    • termsOfServiceUrl：一个url，不知道干什么用的
    • version：可以定义文档的版本
    • contact：联系方式，入参是contact对象、String对象
    • license和licenseUrl：证书

    最后调用build函数构建ApiInfo。

    ¶createRestApi函数：看源码比较好理解

    createRestApi函数返回一个Docket对象。

    • 调用自定义的apiInfo函数，设置api信息。

    • 调用select函数，返回一个ApiSelectorBuilder对象。

    • ApiSelectorBuilder对象可以指定在哪里生成api文档，（apis、paths），入参是一个selector。

    • 比如在哪些包下（basePackage）、哪些类下（declaringClass、withClassAnnotation）、哪些方法下（withMethodAnnotation）。

    • 如果不指定包、类或者方法，就会根据默认进行选择。

      默认是选择所有没打ApiIgnore注释的类和方法，以及全部路径。

    • 最后调用build函数构建。

¶Swagger常见注释

具体内容在io.swagger.annotations、springfox.documentation.annotations等包里。

注释名 特殊属性 用法 示例
@ApiIgnore 添加在方法或类上 忽略这个类或者方法，不展示。
@Api tags、values、hidden 添加在类上，tags打标签代替默认名字 @Api(value = “书籍Controller”,tags = “书籍的CRUD”, hidden = true)
@ApiOperation values、notes、code、response、httpMethod 添加在方法上，values声明方法的作用、notes实施说明，httpMethod声明方法 @ApiOperation(value = “插入书籍”, code = 200, notes = “用于插入书籍”, response = Book.class, httpMethod = “post”)
@ApiImplicitParams 添加在方法上，声明一组参数。 @ApiImplicitParams({ @ApiImplicitParam(name = “book”, value = “书籍类”, required = true, paramType = “json”, examples = @Example(value = @ExampleProperty(mediaType = “json”, value = “测试” )) ) })
@ApiImplicitParam paramType：参数放在哪里dataType：参数类型 添加在ApiImplicitParams里，用于声明一个参数。
@ApiModel subTypes 添加在响应类上，如Book @ApiModel(“书籍对象”)
@ApiModelProperty readOnly 添加在响应类的属性上
@ApiParam 添加在参数前 @ApiParam(name = “书id”, example = “1024”, required = true, value = “书籍id”)
@ApiResponse 添加在ApiResponses里，一般表示一个错误信息。 @ApiResponses({ @ApiResponse(code = 400, message = “请求参数没填好”), @ApiResponse(code = 500, message = “服务器错误”) })
@ApiResponses 添加在请求方法上，表示一组响应

@ExampleProperty 添加在@Example里，表示一个样例。
