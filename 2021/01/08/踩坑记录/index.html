¶idea提示git找不到关联分支？

出现这个问题是本地分支无法和远程分支匹配。

解决办法： 指定该分支的远程分支

¶代码push到错误的远程分支怎么办？

首先不要慌。

先新建一个本地仓库，将已提交的代码保存起来，即将当前分支checkout一份出去。

回退会使这次提交的代码会消失，所以需要保存代码。

使用 git reset head~1 命令回退到上次提交，然后git push -f 强制将本次回退，提交到远程分支。

或者使用git log 命令查看分支的提交记录，它会打印出每一次commit的ID，然后使用命令 git reset --hard 'commit_id' 命令回退到指定的commit状态，注意这里的commit_id是之前查到的commit的ID，然后git push -f 强制将本次回退，提交到远程分支。

¶@Builder注解使无参构造失效？

当我们写完一个DTO或者Entity的属性时，我们不想再写他的get和set方法，就可以使用Lombok的@Data注解，自动生成，当我们不想使用构造函数生成对象时，可以使用@Builder注解来，采用Builder的形式设置属性值获取对象，但是@Builder会使类的默认的无参构造函数失效，导致不能使用无参构造函数来创建对象。

解决办法： 在类上添加两个注解，指定生成构造函数。

  2. 在接口参数上加上@Valid注解，说明要对这个DTO进行校验，同时要给这个接口传个校验结果BingResult。
